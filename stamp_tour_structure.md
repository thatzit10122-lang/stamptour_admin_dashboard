# 스탬프투어 구조 설명 — 코스(Course)가 있는 경우

## 1. 개요

스탬프투어는 사용자가 지정된 장소를 방문하면 스탬프를 획득하고, 일정 개수를 모으면 등급(Grade)을 달성하여 선물을 신청할 수 있는 투어 시스템이다.

**코스가 있는 경우**, 전체 투어는 복수의 독립된 코스로 나뉘며, 각 코스는 서로 완전히 분리되어 운영된다.

---

## 2. 핵심 구조 요소

### 2-1. 코스 (Course)
- 스탬프투어 전체는 **1개 이상의 코스**로 구성된다.
- 각 코스는 서로 **완전히 독립**되며, 코스 간에는 칸막이(barrier)가 존재한다.
- 코스 간 스탬프는 **절대 섞이지 않는다**.
- 사용자는 각 코스를 **독립적으로** 진행한다.

### 2-2. 장소 (Place / Stamp Location)
- 각 코스는 **하나 이상의 장소**로 구성된다.
- 사용자가 특정 장소를 방문하면 해당 코스의 스탬프 1개를 획득한다.
- 한 장소에서 획득한 스탬프는 **그 장소가 속한 코스에서만** 누적된다.

### 2-3. 스탬프 누적 규칙
- 스탬프는 **코스 단위**로 별도 카운트된다.
- 예: 1코스 장소에서 획득한 스탬프 → 1코스 스탬프 수에만 합산
- 2코스, 3코스 등 다른 코스의 스탬프 수에는 영향을 주지 않는다.

### 2-4. 등급 (Grade)
- 등급은 **각 코스 안에서 획득한 스탬프 개수**를 기준으로 결정된다.
- 등급명 규칙: `{코스명}_{달성 스탬프 수}`
  - 예: `1코스_5`, `2코스_9`, `3코스_13`
- 한 코스에 **복수의 등급**이 존재할 수 있다 (단계별 미션).
- 등급 달성 조건(필요 스탬프 수)은 코스마다 다를 수 있다.

### 2-5. 선물 신청
- 특정 등급을 달성하면 **해당 등급에 대한 선물 신청**이 가능해진다.
- 선물 신청 자격은 코스별·등급별로 독립적으로 부여된다.

---

## 3. 데이터 구조 (스키마)

```
StampTour
└── Course[]            ← 복수의 코스
    ├── course_id       (예: "1코스", "2코스")
    ├── Place[]         ← 코스에 속한 장소 목록
    │   ├── place_id
    │   ├── place_name
    │   └── stamp_zone  (위도, 경도, 인식 범위)
    └── Grade[]         ← 코스 내 달성 가능한 등급 목록
        ├── grade_id    (예: "1코스_5", "1코스_10")
        ├── required_stamps  ← 해당 등급 달성에 필요한 스탬프 수
        └── gift_available   ← 선물 신청 가능 여부 (true)
```

---

## 4. 동작 흐름

```
사용자가 장소 방문
        ↓
해당 장소가 속한 코스 확인
        ↓
그 코스의 스탬프 카운트 +1
        ↓
코스 내 누적 스탬프 수 확인
        ↓
달성 가능한 등급 조건과 비교
        ↓
[조건 충족 시] 등급 달성 → 선물 신청 가능
[조건 미충족 시] 계속 진행
```

## 5. 예시

아래는 3개 코스로 구성된 스탬프투어 예시이다.

### 예시 구성

| 코스 | 장소 수 | 등급 조건 |
|------|---------|-----------|
| 1코스 | 10개 장소 (1코스_1장소 ~ 1코스_10장소) | 스탬프 5개 → `1코스_5` 등급 달성<br>스탬프 10개 → `1코스_10` 등급 달성 |
| 2코스 | 9개 장소 (2코스_1장소 ~ 2코스_9장소) | 스탬프 5개 → `2코스_5` 등급 달성<br>스탬프 9개 → `2코스_9` 등급 달성 |
| 3코스 | 13개 장소 (3코스_1장소 ~ 3코스_13장소) | 스탬프 13개 → `3코스_13` 등급 달성 |

### 예시 시나리오

- 사용자가 **1코스** 장소 7곳 방문 → 1코스 스탬프 7개 누적
  - `1코스_5` 등급 달성 → 선물 신청 가능
  - `1코스_10` 등급은 미달성 (3개 부족)
- 동시에 **2코스** 장소 3곳 방문 → 2코스 스탬프 3개 누적
  - 2코스 등급 미달성 (2개 부족)
- 1코스 스탬프(7개)는 2코스 스탬프(3개)에 **전혀 영향을 주지 않음**

## 6. 실제 적용 예시 — 부산 영도구청 스탬프투어

위 구조를 실제로 적용한 사례로, 5개 코스로 구성된다.

| 코스명 | 대표 장소 예시 |
|--------|--------------|
| 영도다리 너머로 | 봉산마을, 삼진어묵, 봉래시장, 봉래동물양장, 영도관광안내센터, 깡깡이예술마을, 영도대교 |
| 영도야경 속으로 | 봉래산 정상, 영도 조내기 고구마 역사기념관, 해돋이마을, 청학배수지전망대, 청학시장, 영도 마리노 오토캠핑장 |
| 중리 바다로 | 태종대 오션 플라잉 테마파크, 영도해녀문화전시관, 중리노을전망대, 75광장, 하늘전망대 |
| 해양역사 속으로 | 영도공간 307, 태종대, 감지해변, 동삼동패총전시관, 아미르공원, 국립해양박물관, 동삼해수천 |
| 흰여울 바다로 | 흰여울문화마을, 절영해안산책로 입구, 남항시장, X-SPORTS 광장, 남항등대 |

각 장소에는 GPS 기반 스탬프존(위도, 경도, 인식 범위)이 설정되어 있어, 사용자가 해당 반경 내에 위치할 때 스탬프가 인식된다.

---

## 7. 핵심 규칙 요약

1. **코스 간 스탬프는 절대 공유되지 않는다.**
2. **등급은 코스별로 독립적으로 산정된다.**
3. **하나의 코스에 복수의 등급(단계)이 존재할 수 있다.**
4. **등급 달성 시 해당 등급에 대한 선물 신청 권한이 부여된다.**
5. **코스 내 모든 장소를 방문하지 않아도, 조건 충족 시 등급 달성이 가능하다.**
